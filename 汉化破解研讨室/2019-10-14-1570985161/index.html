 【脑残作死异想天开】论PS3版Date a Live移植到PC平台的可能性

总而言之，谁来帮我搞定MP动态立绘吧，这样我们还可以继续愉快的玩耍……




首先，这篇帖子的前提是Leaf社移植的PC版WHITE.ALBUM，然后有朋友偶然提起过DAL移植，虽然可能只是玩笑，不过我留意了一下，然后就逗逼的下载了下面提到的游戏，发现这三个游戏所用的引擎是同一个，
文件结构也异常相似，PS3和PC版的WA文件基本上是一模一样的，把PS3WA的封包直接覆盖PC版的同名文件，游戏也可以正常运行。于是我就尝试了下，能不能用DAL的文件替换PCWA的，不过…………

写在前面：我觉得看这贴的人不要觉得我是纯扯蛋，PC平台的游戏，有素材就可以通过ONS移植到手机，那PS3的为何不能移植到PC呢？
而且实际上对于DAL的素材，就差这一个动态立绘，其他的都可以破解。

一、涉及到的游戏：
帖子是基于这三个游戏进行的：
1、PC版WHITE.ALBUM－缀られる冬の想い出－（PCWA）
2、PS3版DATE·A·LIVE 约会大作战凛祢ユートピア （PS3DAL）
3、PS3版WHITE.ALBUM－缀られる冬の想い出－（PS3WA）

二、可能的两种移植方式：
能够想到的也只有这两种方式了，各有利弊。

1、移植核心：KRKR引擎
    如果能把PS3DAL的所有游戏要素（movie sound image text）全都提取出来，那么只要有一名或几名经验丰富的krkr程序员就可以完成移植。优点在于不用去分析游戏的脚本流程和一大堆opcode。
但这里有个问题，也是两种方法共同的问题：这三个游戏都用到了MP动态立绘技术，这玩意看起来略复杂……这也是我碰到的难点。

2、移植核心：PCWA的主程序
    这个程序不能算是引擎，因为里面对WA做了许多针对性的限定，而不是完全通过外部文件决定游戏进程。
但既然它能读PS3的文件，两游戏的文件结构又及其相似，我觉得还是有移植的可能性的。只要能让它读取PS3DAL的立绘，那么其他问题就有可能通过逆向解决。

三、下面说说具体的文件分析：

     澄空推荐的图床速度有点慢，有的同学可能有图看不到，看不到的到这里找：
     http://tieba.baidu.com/p/3083249463
     至于贴吧里图片的顺序我就不调整了……

    1、封包：
          这三个游戏用的都是pck封包，这个封包格式并不难，解包工具可以从asmodean下载，有源码，就不多说了。下面的工具包里有我修改过的源码，增加了封包功能。

    2、视频：
          PS3游戏的视频是pam文件，里面包括了h264编码的视频和atrac3+编码的音频，用vgmtoolbox(http://sourceforge.net/projects/vgmtoolbox/files/?source=navbar)
          可以同时提取视频流和音频流。

          PC游戏的视频虽然也是pam扩展名，但其实质是wmv3(Windows Media Video 9)编码的wmv视频。
          
          两种视频都可以直接用potplayer播放，但是前者会放不出声音。这没有关系，可以用ffmpeg进行简单的转码和音视频合成。

    3、图片：

          PC版的图片使用了一种lz77压缩，我不知道是不是变种，也没仔细分析，直接内联汇编了Orz……解出来的是bmp的所有象素数据，丢到libpng里就可以直接生成png图片。
          PS3版的图片就更简单了，直接就是png文件加上个Texture文件头。
          用伪压缩方法我写了ps3tex2pctex图片转换工具。源码也放在后面工具包了。

          这里还有一点要说的是，很多图片都是一张大图里面包括了许多小图，这样的图在文件尾部有Part索引来记录每个part在原图的位置和在屏幕上显示的位置：

Copy code
struct PARTHDR {
  unsigned char signature[8]; // "Pack    "
  unsigned long index_size;
  unsigned long part_count;
};

struct PARTENTRY {
  //每个part在游戏屏幕中的长宽
  float screen_x2;
  float screen_y2;
  //每个part在整张图片中左上角坐标的因数，乘以整张图的长或宽为实际坐标
  float pic_mult_x1;
  float pic_mult_y1;
  //每个part在整张图片中右下角坐标的因数，乘以整张图的长或宽为实际坐标
  float pic_mult_x2;
  float pic_mult_y2;
};

      5、脚本：
            脚本部分我也没分析，这也是移植的难点之一。
            不过只是文本的话，PCWA是明文，PS3DAL的文本也有办法提取，毕竟汉化都开坑了……
            文本不是难点，难点是脚本文件(*.sdat)里不只有文本，还有opcode，来调用游戏其他资源和控制流程，这部分的分析将是痛苦的，甚至是不可能的……

            和官方例程对比加上官方手册，加上一些推测：
