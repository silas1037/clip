} LOGFONT;


この構造体のメンバーlfFaceNameはフォントのタイプフェイス名前を指定する文字列です。
OllyDBGでの場合は、コンマンドバーに「d 12FBCC + 1C 」を入力するとフォントの名前が表示されます。

Copy code
int EnumFontFamiliesEx(

    HDC hdc,    // handle to device context
    LPLOGFONT lpLogfont,    // pointer to logical font information
    FONTENUMPROC lpEnumFontFamExProc,    // pointer to callback function
    LPARAM lParam,    // application-supplied data
    DWORD dwFlags    // reserved; must be zero
    );    
  

Parameters

hdc

Identifies the device context.

lpLogfont

Points to a LOGFONT structure that contains information about the fonts to enumerate. The function examines these members:

Member    Description
lfCharset    If set to DEFAULT_CHARSET, the function enumerates all fonts in all character sets. If set to a valid character set value, the function enumerates only fonts in the specified character set.
lfFaceName    If set to an empty string, the function enumerates one font in each available typeface name. If set to a valid typeface name, the function enumerates all fonts with the specified name.
lfPitchAndFamily    Must be set to zero for all language versions of the operating system except Hebrew and Arabic. For these languages, set IfPitchAndFamily to MONO_FONT to enumerate only fonts that provide all codepage characters within the font.
 

lpEnumFontFamExProc

Points to the application-defined callback function. For more information about the callback function, see the EnumFontFamExProc function. 

lParam

Specifies a 32-bit application-defined value. The function passes this value to the callback function along with font information. 

dwFlags

Reserved; must be zero.


Copy code
int CALLBACK EnumFontFamExProc(

    ENUMLOGFONTEX *lpelfe,    // pointer to logical-font data
    NEWTEXTMETRICEX *lpntme,    // pointer to physical-font data
    int FontType,    // type of font
    LPARAM lParam    // application-defined data 
   );    
 

Parameters

lpelfe

Points to an ENUMLOGFONTEX structure that contains information about the logical attributes of the font. 

lpntme

Points to a structure that contains information about the physical attributes of a font. The function uses the NEWTEXTMETRICEX structure for TrueType fonts; and the TEXTMETRIC structure for other fonts. 
Windows 95: The NEWTEXTMETRICEX structure is not implemented. Use NEWTEXTMETRIC instead.

FontType

Specifies the type of the font. This parameter can be a combination of these values:
DEVICE_FONTTYPE
RASTER_FONTTYPE
TRUETYPE_FONTTYPE

lParam

Specifies the application-defined data passed by the EnumFontFamiliesEx function.


概説的な叙述のように、実際のタイプフェイス名前はlpEnumFontFamExProcを指定して関数に渡されます。即ち、引数として渡された構造体「ENUMLOGFONTEX 」の中身はタイプフェイス名前が含まれます。

Copy code
The ENUMLOGFONTEX structure contains information about an enumerated font.

typedef struct tagENUMLOGFONTEX {
    LOGFONT  elfLogFont;
    BCHAR  elfFullName[LF_FULLFACESIZE];
    BCHAR  elfStyle[LF_FACESIZE];
    BCHAR  elfScript[LF_FACESIZE];
} ENUMLOGFONTEX;
  

Members

elfLogFont

Specifies a LOGFONT structure that contains values defining the font attributes.

elfFullName

Specifies a null-terminated string specifying the unique name of the font. For example, "ABC Font Company TrueType Bold Italic Sans Serif".

elfStyle

Specifies a null-terminated string specifying the style of the font. For example, "Bold Italic".

elfScript

Specifies a null-terminated string specifying the script of the font. For example, "Cyrillic".


標的はその「elfFullName」っていうメンバーです。コールバック関数の中身を確認すると必ず見つけることができます。

EnumFontFamilies还是EnumFontFamiliesEx？LZ大大要的是后者的样子，看参数表是后者的。EnumFontFamiliesEx修改的关键与CreateFontIndirect一样，要跟上LOGFONT的内容，关注点是FaceName和CharSet。这两个值是EnumFontFamiliesEx的判断标准，LOGFONT里的其它值由EnumFontFamiliesEx赋值。所以关键就是跟到这LOGFONT初始赋值的地方。而callback function里的内容不是最关键的地方，但也值得关注。

在调用EnumFontFamiliesEx之前，作为参数的LOGFONT只需要对这几个成员赋值：
lfCharset，lfFaceName，lfPitchAndFamily
且不是3个都必须要赋初值，只要处理需要的，例如小的只要限制语系，那就只给Charset赋值。
LOGFONT内的其它值会由EnumFontFamiliesEx写入。调用EnumFontFamiliesEx就是为了枚举字体名，当然就不需要先把具体的字体名作为参数给到EnumFontFamiliesEx里。用F8步过EnumFontFamiliesEx的调用，看看LOGFONT里的数据发生了什么变化就知道。
