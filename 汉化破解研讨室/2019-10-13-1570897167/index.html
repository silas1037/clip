 [转火星帖]灯穂奇譚的文件格式

闲逛才发现原来CK有专门的汉化技术区.不自量力,把暑假的时候弄过的点东西转过来凑数 XD
原地址: http://www.ahcomic.com/bbs/viewthread.php?tid=132160


官网: http://www.mizu-kagami.jp/tousui/tousui_index.html
某评论: http://tubaumi.exblog.jp/756756

由「水鏡」所开发的同人游戏，『灯穂奇譚』，其游戏引擎虽然不是制作同人visual novel常见的免费引擎而是自行编写的，却也是十分典型的一个，基本功能齐全，算是个精简的实现吧。字体之类虽然不能在菜单里直接调，也罢，问题不大。
还是得赞叹，一个同人能做到这水平真是不错。

有个朋友说想汉化，于是帮忙看了看文件格式。这游戏就同那位朋友说的一样，相当的“厚道”。下面详细说明。

主要打包（archive）格式为AOD格式，后缀为aod。该格式是比较naive的打包格式，无加密无压缩，只是对原始内容做了索引打包起来；有16字节对齐。文件被分为段，每段的结构是：

- 32字节索引 × 30（第一段里是31个）
- 32字节指向下一段的指针（若直到该段末尾仍未能把所有文件打包，则此部分存在）
- 原始文件内容

由此可知，每段可以容纳30个文件（第一段里是31个），放不下的另开一段放，直到所有需要打包的文件都放下为止。

让我们来看看每个索引的结构：

其中前16字节（绿色部分）是为文件名预留的空间，偏移量0x28-0x2B的4个字节（黄色部分）是段内偏移相关数据，偏移量0x2C-0x2F的4个字节（蓝色部分）是原始文件大小。这些数字都是32位整型，对齐方式是little-endian。索引部分还有些另外8个字节的数据（红色和橙色部分），暂时不了解其用途；这些看似随机填充的数据，全部改为00后游戏仍然能正常运行，故判定正常情况下应该不需要这些数据，可能只是某种标记。
关于黄色部分需要特别说明。这里指明了是段内偏移相关数据而不直接是段内偏移量。
  实际的段内偏移量 = 段起始地址 + 黄色部分数据 + 0x40 × 段内序号（从1开始）
相当奇怪的偏移量记录方式。

整个AOD文件中，除最后一个段外，其他段都会有指向下一段的指针：

这个部分，白色区域都是固定的，只有紫色部分这4个字节会改变，记录的是本段中原始文件内容所占用的总大小，也就是从内容部分的首地址开始到下一段首地址间的偏移量。

接下来让我们来看看原始文件内容被打包后，其尾部的样子：



这两幅图中，绿色部分是属于原始文件的内容，红色部分是填充内容。
可以看到，在16字节对齐的要求下，若原始文件的尾部如果不能满足16字节对齐，则对不足空间进行填充；若刚好满足16字节对齐，则填充16字节的无效数据。填充数据的选择让人感到有点困惑，经测试，任意垃圾数据填入都没问题。

AOD打包格式基本特征就这么多。这里的描述足够构造一个打包/解包程序了。
已经写好了打包/解包程序的原型，而且也已经把需要的东西都提取出来了。不过实在不是什么难事，那代码就不专门拿出来献丑了。

其中magic value指的是跟偏移量相关的值。通过比对可知该值与实际地址以及段基地址之间的关系。

该游戏中其他文件格式的使用状况：

图片：PNG BMP
动画：MPEG
音频：OGG Vorbis
脚本：纯文本

都是业界非常常用的格式，没有做特殊处理。BMP有24位和32位的。
另有一未知具体作用的AIM格式。每个AIM文件里都包含了1个文件名（预留了16字节空间），和2个有完整bitmap头信息却没有足够数据的数据段。估计是某种压缩了的带透明度的bitmap。

使用了DirectX8的支持。

一如其他常见的日本制visual novel，该游戏所使用的文字编码是Shift-JIS，而且编码一旦改变就报错。脚本所使用的是自定义的脚本语言，有部分面向对象程序语言的特征。游戏实际使用的脚本的注释相当完整，无须对脚本多做分析就能了解其意义和使用方法，也方便了对游戏的再制作。编码问题的解决方案，除了装钩子做点手脚外，我暂时没什么别的好想法。如果有人有什么别的实践过的好办法的话，请务必赐教 ^ ^

需要翻译的文本大概是27万多字。

【举手】我想问一下，提取出来的资源当中有一些是aim后缀的，请问那些文件是怎么回事？要怎么查看？谢谢~~
顺便附上我的源代码，稍微有点挫...

#include<iostream>
#include<string>
#include<fstream>
#include<iomanip>
#include<stdio.h>
#include<direct.h>
using namespace std;

#define byte unsigned char
struct file_info
{
    char name[40];
    int mag_val;//段内偏移地址，要加上k*0x40才算真正的
    int size;
    int segment;//段起始地址
    int k;
    int addr;
    int addr_calc()//文件内容的真正地址
    {
        addr=segment+mag_val+k*0x40;
        return addr;
    }
};
char *tmp=NULL;

//文件结构——
//- 64字节索引 × 30（第一段里是31个）
//- 64字节指向下一段的指针（若直到该段末尾仍未能把所有文件打包，则此部分存在）
//- 原始文件内容

void createDir(string str)
{
    int k=str.size();
    while(str[k-1]!='\\')
        k--;
    str[k]=NULL;
    _mkdir(str.c_str());
}

int main()
{
    ifstream ifile;
    ifile.open("G:\\game\\灯穂奇譚\\aod\\img0.aod",ios::binary);
    file_info *f=new file_info[1000];
    int file_num=1,k=1,seg=0;

    ifile.read(f[0].name,40);
    ifile.read((char*)(&f[0].mag_val),sizeof(f[0].mag_val));
    ifile.read((char*)(&f[0].size),sizeof(f[0].size));
    f[0].k=k++;
    f[0].segment=seg;
    ifile.seekg(16,ios::cur);

    while(true)
    {
        for(int i=0;i<30;i++)
        {
            ifile.read(f[file_num].name,40);
            ifile.read((char*)(&f[file_num].mag_val),sizeof(f[file_num].mag_val));
            ifile.read((char*)(&f[file_num].size),sizeof(f[file_num].size));
            f[file_num].k=k++;
            f[file_num].segment=seg;
            ifile.seekg(16,ios::cur);
            if(f[file_num].mag_val!=0)
                file_num++;
        }
        ifile.seekg(40,ios::cur);
        int next_seg;
        ifile.read((char*)(&next_seg),sizeof(next_seg));
        if(next_seg==0)
            break;
        seg+=next_seg+0x40*k;
        k=1;
        ifile.seekg(seg,ios::beg);
        //输入各段其实地址，方便检查
        cout<<setiosflags(ios::left|ios::uppercase);//输出左对齐、字母大写
        cout<<setw(20)<<dec<<seg<<hex<<"0x"<<setw(0)<<seg<<endl;
    }
        
    for(int i=0;i<file_num;i++)
    {
        //cout<<setw(20)<<f.name<<setw(10)<<f.addr_calc()<<endl;
        ofstream ofile;
        string str="G:\\game\\灯穂奇譚\\aod\\img0\\";
        str.append(f.name);
        createDir(str);            //创建目录（如果文件名中自带一个子目录的话）
        ofile.open(str,ios::binary);
        tmp=new char[f.size];
        ifile.seekg(f.addr_calc(),ios::beg);
        ifile.read(tmp,f.size);
        ofile.write(tmp,f.size);
        ofile.close();
        delete []tmp;
    }
    delete []f;
    ifile.close();
    cout<<"finished"<<endl;
    return 0;
}
