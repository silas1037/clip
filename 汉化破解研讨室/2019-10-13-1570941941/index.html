那个4字节是name_hash

游戏按照最后4字节的name pointer找到name 然后调用下面这个函数转换为name hash，这个hash用于快速搜索资源的。

static DWORD get_hash(char *name)
{
    int i = 0;
    unsigned int hash = 0;

    for (int chr = *name++; chr; chr = *name++) {
        chr -= 32;
        chr &= 0xff;
        chr <<= i;
        i += 5;
        if (i >= 25)
            i = 0;
        hash += chr;
    }

    return hash % 0xfff1;
}

这个是name到hash的变换算法 不过这个hash算法质量不好 很多资源hash重复 所有才有最后4字节的那个name pointer： 当hash一致时再比较资源名称从而唯一确认要读取的资源。

005A5F11 |. 8B7424 08 mov esi,dword ptr ss:[esp+8] ; esi得到文件名的首地址

005A5F18 |. 33C0 xor eax,eax ; eax=0
005A5F1A |. 33C9 xor ecx,ecx ; ecx=0
005A5F1C |. 46 inc esi ; esi指向第二个字节
005A5F1D |. 85D2 test edx,edx ; 如果第一个字节=0则不作循环处理
005A5F1F |. 74 1F je short YSO_WINb.005A5F40
005A5F21 |> 83C2 E0 /add edx,-20 ; edx=edx-20
005A5F24 |. 81E2 FF000000 |and edx,0FF ; edx=edx & FF
005A5F2A |. D3E2 |shl edx,cl ; edx=edx<<cl

005A5F2F |. 03C2 |add eax,edx ; eax=eax+edx
