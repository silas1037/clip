 求助加密插件的编写

管理提醒：
本帖被 ZOKKON 从 【CK-GAL中文化】 移动到本区(2008-03-02)
在做SWAN SONG的汉化工作.
解包工作已经解决了.
用crass-0.4.7.17和kirikiri插件用xror参数可以提取素材.
但制作补丁的话还要对素材用同一方式进行加密才能被程序读取.
求一个加密程序或者xp3enc.dll加密插件
xp3dec.tpm下载地址:http://pickup.mofile.com/2850402839970326
补充游戏的解密方式,感谢赤血挽歌提供：
static void xp3filter_decode_xor_ror(BYTE *buf, unsigned int len, DWORD hash)
{
        BYTE shift, xor;

        shift = (BYTE)hash;
        xor = (BYTE)((hash >> 8) & 0xff);
        if (!shift)
                shift = 15;
    if (!xor)
                xor = 0xf0;

        shift &= 7;
        for (unsigned int i = 0; i < len; i++) {
                buf ^= xor;
                buf = (buf >> shift) | (buf << (8 - shift));
        }
}

在你的这个帖子里看到了解密的代码
如果按你介绍的代码来看，解密的过程就是先作了一次xor运算，再作一次shift运算。
那么加密过程自然就是先来一次shift，再来一次xor就好了。
何必写什么插件，直接把他们写到打包代码里就好了，也就两行代码而已，省了dll的调用时间，也方便调试。
