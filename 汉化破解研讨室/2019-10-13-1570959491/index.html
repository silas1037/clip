编码边界检查有问题，以上BZ提到的3处全部将范围改成了81~FE，EO~FE，运行，“□”呈上。

要正常显示中文，需修改两处地方。

1.CreateFontA 的参数

CreateFontA 有两处调用，0x4BA642 和 0x4BA57D
它们共用同一个 iCharSet  参数。
EXE偏移：0x206718

原值：
Copy code
.rdata:00607914 ; DWORD iCharSet 
.rdata:00607914 iCharSet dd 1 
.rdata:00607918 db 80h

修改：
Copy code
.rdata:00607914 ; DWORD iCharSet 
.rdata:00607914 iCharSet dd 1 
.rdata:00607918 db 86h



2.边界检查

地址：0x4B8F85
EXE偏移：0xB8385

原值：
Copy code
 
.text:004B8F83 mov al, dl 
.text:004B8F85 xor al, 20h 
.text:004B8F87 add al, 5Fh 
.text:004B8F89 cmp al, 3Ch 
.text:004B8F8B sbb ecx, ecx 
.text:004B8F8D neg ecx 
.text:004B8F8F cmp ecx, 1 
.text:004B8F92 jnz short loc_4B8FA1


修改：
Copy code
 
.text:004B8F83 mov al, dl 
.text:004B8F85 cmp al, 81h 
.text:004B8F87 jbe short loc_4B8FA1 
.text:004B8F89 nop 
.text:004B8F8A nop 
.text:004B8F8B nop 
.text:004B8F8C nop 
.text:004B8F8D nop 
.text:004B8F8E nop 
.text:004B8F8F nop 
.text:004B8F90 nop 
.text:004B8F91 nop 
.text:004B8F92 nop 
.text:004B8F93 nop


上面的修改是把边界检测由“日文范围字符”改为“大于等于0x81的字符”都作为双字节字符处理。
