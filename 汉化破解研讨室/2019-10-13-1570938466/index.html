主要看一下div / 7这句话的反汇编是什么。
请按这个格式注明（debug版的不用贴了，肯定是DIV了，如果被强制内联变成直接push 7的就都算了）：

编译器：vc6
反汇编：

引用 编辑 短信 举报 只看 锚点

 

这应该是所谓的倒数乘法，因为IA-32结构中乘法比除法快4~6倍，所以编译器经常用倒数除法处理除数为常数的除法。

除以7的情况：
Copy code
MOV    ECX, DWORD PTR SS:[ESP+4]  ; 参数div
MOV    EAX, 24924925
MUL    ECX
MOV    EAX, ECX
SUB    EAX, EDX
SHR    EAX, 1
ADD    EAX, EDX
SHR    EAX, 2

可以这样转化：
Copy code
y = ((x - x * sr) / 2 + x * sr) / 4


其中x为被除数，sr = 0x24924925
sr * 7 = 0x100000003 = 0x100000000 + 3
(sr - 1) * 7 = 0xFFFFFFFC = 0x100000000 - 4

可以将x * sr理解成“定点小数乘法”，即小数点位置固定的实数的乘法。从上面的式子不难看出sr为 (1/7) * 0x100000000，使用mul指令后，高位(edx)保存的就是x * (1/7)的整数部分，低位(eax)就是小数部分。因此上面的代码可以看做是
Copy code
y = ((x - x/7) / 2 + x/7) / 4 = x/7

至于为什么不直接y = x * sr，是因为1/7精度有限，多次运算累计误差会很大，因此使用减法运算抵消误差。
