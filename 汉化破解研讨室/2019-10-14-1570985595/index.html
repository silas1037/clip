這個方法只針對有 .detour 區段的處理
進階的 IAT 還原請自己研究，這裡不負責了

1. 使用 odbg201+renameod+scyllahide 過殼驗證
2. 用 alpharomdie_20121001 版本對 Anoore.exe 破解 (有kDays.dll的)
3. 跑起來之後對 .text 區段搜尋指令「mov r32,bb40e64e」(VC++編譯的初始化代碼特徵)
4. 這個 call 的起點在 4906d1，再次搜尋「call 4906d1」(快捷鍵Ctrl+F)
5. 結果定位到 4871a3，這個就是 oep 了，下個硬體斷點重跑一次
6. 自己想辦法 dump 出來後，hex dump 定位到 anoore.exe 的 .detour 區段
7. 顯示格式切換到 integer - address 方便觀察
    +10 是 Import Directory RVA
    +14 是 Import Directory Size
8. 找個 pe 修改工具改改，.text 區段的 rwe 權限別忘了改
