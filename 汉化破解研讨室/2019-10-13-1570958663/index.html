 DC系列的程序修改的补充和免序列号的方法

嗯，因为有人要我做这个东西所以就动手了，顺便写个教程。
因为乙津梦版主的帖子有一个非常重要的一点没说到所这贴只是作为补充而存在的，前期过程请参考乙津梦版主的帖子。

所谓的DC系列错别字问题其实发生的原因是这样的：
由于这个游戏是用API中的GetGlyphOutlineA函数输出，所以可以很好理解每个字在从脚本读进来之后都是点阵图的形式。
而这个游戏的字符输出时是大约每788个字节保存一个字符的点阵图。以上不算重要可以忽略。
然后从脚本里读字符进来之后，这个游戏并不是和某些游戏一样内置有字符的点阵图，所以需要即时演算生成。
也就是说当读入一个字之后，才会生成一个点阵图。
比如“鲜花盛开”这四个字，就会生成四张点阵图。
但是“花开花落”这四个字，是生成四张点阵图么？
答案是：不是，是三张。
因为“花”出现了两次，在第二次从脚本中读取到“花”字的时候，这个游戏由于检测到“花”这个字已经出现过，所以便会进入点阵图区寻找，然后直接调用那幅图显示出来，不用重新生成。

然而在修改了了CREATEFONTA函数和边界检查后，却郁闷的发现，某些字显示的不正确……
比如这句“无论开放时多么美丽的花朵”在游戏中却显示成“无论开放时多么么丽的花多”。
后来研究了一下显示的不正确的字的特点后才发现，之所以 美->么 ，朵->多 ，不光是它们读音相似233.
而是因为：它们的GBK码高位相同。
比如：
么的GBK码是C3 B4
美的GBK码是C3 C0
还记得我前面说的相同字符匹配么？好死不死就是在修改程序后相同字符匹配变成只匹配高位了- -，所以说，只要是高位相同的字符显示出来就全都是第一个出现的字符的样子。
于是就产生了这个问题。解决方法也很简单，将匹配语句去掉即可，不过我是将匹配语句后面的跳转语句去掉了

Copy code
004069A8  |.  8BF8          mov     edi, eax
004069AA  |>  33C0          xor     eax, eax
004069AC  |.  B9 F8BC6200   mov     ecx, 0062BCF8
004069B1  |>  8B11          /mov     edx, dword ptr [ecx]
004069B3  |.  3BD7          |cmp     edx, edi
004069B5      74 61         |je      short 00406A18
004069B7  |.  85D2          |test    edx, edx
004069B9      74 74         |je      short 00406A2F
004069BB  |.  81C1 14030000 |add     ecx, 314
004069C1  |.  40            |inc     eax
004069C2  |.  81F9 F8D06500 |cmp     ecx, 0065D0F8
004069C8  |.^ 7C E7         \jl      short 004069B1

其中
Copy code
004069B3  |.  3BD7          |cmp     edx, edi

即为判断是否相等的语句，EDI中存放的是待处理字符的高位，EDX中存放的是已出现的字符的高位。
Copy code
004069B5      74 61         |je      short 00406A18

这句就是跳转，不要犹豫用NOP填充掉它吧~~

接下来是免序列号的修改方法：

进入OLLYICE载入DC2FL后，CTRL+N查看当前模块中的所有名称。
可以看到其中有一个API是DialogBoxParama,右击，选择在每个参考上设置断点。（因为DC系列序列号验证是弹出对话框，所以我猜测在对话框有关的API上入手应该有效果）
然后按F9运行程序。
这样等到出现需要输入序列号的对话框时程序就会停下来。
程序停下的地方：
Copy code
0040A720  /$  A1 1840A200   mov     eax, dword ptr [A24018]
0040A725  |.  8B0D E43BA200 mov     ecx, dword ptr [A23BE4]          ;  DC2FLCN.00400000
0040A72B  |.  6A 00         push    0                                ; /lParam = NULL
0040A72D  |.  68 30A64000   push    0040A630                         ; |DlgProc = DC2FLCN.0040A630
0040A732  |.  50            push    eax                              ; |hOwner => 000D04F8 ('D.C.嘦 Fall in Love丂乣僟丒僇...',class='DC2FL_CLASS')
0040A733  |.  6A 6B         push    6B                               ; |pTemplate = 6B
0040A735  |.  51            push    ecx                              ; |hInst => 00400000
0040A736  |.  FF15 9C414300 call    dword ptr [<&USER32.DialogBoxPar>; \DialogBoxParamA<<<<<<<<<<<<<<<<<<<<<<<<<<<这里
0040A73C  |.  83F8 01       cmp     eax, 1
0040A73F  |.  75 0D         jnz     short 0040A74E
0040A741  |.  6A 00         push    0
0040A743  |.  E8 D8AF0000   call    00415720
0040A748  |.  83C4 04       add     esp, 4
0040A74B  |.  33C0          xor     eax, eax
0040A74D  |.  C3            retn
0040A74E  |>  83C8 FF       or      eax, FFFFFFFF
0040A751  \.  C3            retn

很明显这是一个子函数，对话框就是从这里调用的。但是子函数中貌似没有符合序列号判别的语句，所以应该是在调用前判别序列号然后按照结果来决定是否调用这个子函数。
我们看看它是从哪里被调用的

点击子函数第一行，右击，点击转到——》CALL来自00423460
可以看到
Copy code
00423460  |> /E8 BB72FEFF   /call    0040A720
00423465  |. |3BC6          |cmp     eax, esi
00423467  |. |74 1D         |je      short 00423486
00423469  |. |6A 6C         |push    6C
0042346B  |. |68 A84DA500   |push    00A54DA8                        ;  ASCII "DAGABA8MGOE"
00423470  |. |E8 7B2B0000   |call    00425FF0
00423475  |. |83C4 08       |add     esp, 8
00423478  |. |85C0          |test    eax, eax
0042347A  |.^\7C E4         \jl      short 00423460

如果没有输入序列号，就会在这个循环里出不去（ASCII里那个是我故意修改的错误序列号）
我们往上面看几行，可以看到
Copy code
00423450  |.  E8 9B2B0000   call    00425FF0
00423455  |.  83C4 08       add     esp, 8
00423458  |.  85C0          test    eax, eax
0042345A    ^ 0F8D 7AFEFFFF jge     004232DA

这里的CALL就是用来调用判别函数的，当然我们不用那么麻烦跟进去修改它。多试几次就可以发现如果序列号通过了，那么最后一句的JGE跳转就会被实现的，但是如果没有通过则会进入下面的调用对话框的程序中去。
所以将判断跳转
Copy code
0042345A    ^ 0F8D 7AFEFFFF jge     004232DA

改成强制跳转
Copy code
0042345A    ^\E9 7BFEFFFF   jmp     004232DA
0042345F      90            nop

即可实现免序列号
